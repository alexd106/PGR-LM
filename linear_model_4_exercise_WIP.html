Exercises


Linear models - Model selection


Model selection is one of the most debated topics in applied statistics. Before you start this exercise it is worth being clear about what kind of analysis you are attempting. There are broadly three situations where models are compared or simplified:

  • Confirmatory testing: you had a specific hypothesis before seeing the data (for example, “does grazing intensity affect bird abundance after accounting for patch area?”). In this case you fit the pre-specified model and test it once. Model selection is not needed.
  • Exploratory variable selection: you have several candidate predictors and want to identify which ones contribute meaningfully to explaining the response. This is the situation in the present exercise, and it is where backward elimination is commonly used.
  • Predictive modelling: you want the model that will generalise best to new data. This requires different methods (cross-validation, regularisation) that are beyond the scope of this course.

In practice many analyses blend these situations, and this is where care is needed.


In the previous exercise you fitted a pre-specified model which included the main effects of patch area (LOGAREA), grazing intensity (FGRAZE) and their interaction (FGRAZE:LOGAREA). Here we revisit those data and ask whether a more complete model, incorporating the remaining explanatory variables, is warranted. We will build a maximal model including all main effects and the LOGAREA:FGRAZE interaction, and then simplify it systematically. We focus on only the LOGAREA:FGRAZE interaction because we have relatively limited data (67 observations) and fitting additional interaction terms would require estimating many parameters from few observations. This is a balance you will need to maintain with your own data.


It is also important to note that we will assume all the explanatory variables were collected because the researchers believed them to be biologically relevant for explaining bird abundance. What you should try to avoid is collecting large numbers of variables simply because you can, and then expecting your statistical models to make sense of it all for you. Your biological knowledge must be woven into this process at every step.


What can go wrong with backward elimination

Backward elimination is a useful tool but it carries real risks. Bear these in mind throughout this exercise.

  • Capitalising on chance: at each step you are searching for the least significant term. Across many steps, some terms will appear significant or non-significant purely by chance. The p-values from the final selected model are not valid in the usual sense.
  • The final model is not “the correct model”: it is the most parsimonious description of these data given the candidate variables and the selection criterion used. A different starting model, or a different dataset from the same population, could yield a different final model.
  • Correlation between predictors: if two predictors are correlated, removing one may change the apparent significance of the other. Check for collinearity before you start.
  • Biology must guide every step: a term should not be retained purely because it is statistically significant, nor discarded purely because it is not. Ask at each step whether the decision makes biological sense.


1. Import the loyn.txt data file into RStudio and assign it to a variable called loyn. Here we will be using all the explanatory variables to explain the variation in bird abundance. Do any variables need log-transforming or converting to a factor? Add the transformed variables to the loyn dataframe as you have done in previous exercises.


2. Create a pairs plot of your variables of interest using pairs() and add correlation coefficients to the lower triangle. Pay attention to pairs of explanatory variables with |r| > 0.7, as this indicates potential collinearity that may cause problems when both variables are included in the same model. Note any pairs that concern you.


3. Pairwise correlations do not capture multicollinearity, where one predictor is a near-linear combination of two or more others. A more formal measure is the variance inflation factor (VIF). Fit a model with all main effects only (no interaction term yet) and use car::vif() to compute VIFs for each predictor. VIF values below 5 are generally considered acceptable; values above 10 indicate severe collinearity. What do you conclude?

Note: we check VIFs on a main-effects model because car::vif() does not work sensibly when interaction terms are present.


4. Before fitting the maximal model, take a moment to think about the biology. Based on what you know (or can reasonably infer) about bird communities in forest patches, which of the six explanatory variables would you expect to have the strongest relationship with bird abundance, and which might you expect to have little or no effect? Jot down a brief justification for each variable.

The six variables are: LOGAREA, FGRAZE, LOGDIST, LOGLDIST, YR.ISOL, ALT.

There are no right or wrong answers here. The point is to engage your biological knowledge before you look at any model output, so that the statistics inform your reasoning rather than replace it.


5. Now fit the maximal model. Include all six explanatory variables as main effects and add the interaction LOGAREA:FGRAZE. Call this model M1.


6. Examine the model using summary(). You will find 14 parameter estimates and a corresponding set of p-values. Do not try to interpret each one individually at this stage. Instead, note the overall picture: how much of the variation in ABUND does this model explain (adjusted R-squared), and does there appear to be scope for simplification?


Section A: F-test backward elimination


We will now use drop1(test = "F") to test the contribution of each term. Recall from the previous exercise that unlike anova(), which uses sequential (Type I) sums of squares where the order of terms matters, drop1() tests each term as if it were entered last. This removes any dependence on the order of terms in the model formula and is the appropriate tool for models with multiple predictors.

At each step we remove the term with the largest p-value (least significant), but only if we are comfortable with that decision on biological grounds as well as statistical ones.


7. Run drop1(M1, test = "F"). Which term is the weakest candidate for the model? Before removing it, consider whether there is a biological reason to retain it regardless of its significance. Remove it and call the updated model M2.


8. Run drop1() on M2. Identify the weakest term. Is removing it biologically reasonable? Update the model and call it M3.


9. Run drop1() on M3. Identify the weakest term, consider whether removing it is biologically reasonable, and update the model.


10. Run drop1() on M4. Identify the weakest remaining term, consider it biologically, and update the model.


11. Run drop1() on M5. What happens, and does the model selection process end here?


12. You will have noticed that drop1() does not show p-values for the main effects LOGAREA and FGRAZE in the final model. Why is this?


Section B: AIC-based selection


F-test backward elimination requires a significance threshold (typically p > 0.05 for removal). This threshold is arbitrary, and, as noted above, the p-values at each step are not strictly valid because the same data are used to decide which terms to test. An alternative is to use the Akaike Information Criterion (AIC), which does not rely on p-values. AIC rewards goodness of fit and penalises complexity (number of parameters), selecting the simpler model whenever removing a term does not substantially increase AIC.

A widely used rule of thumb: if removing a term increases AIC by less than 2 units, the evidence for retaining it is weak and the simpler model is generally preferred.


13. Starting again from M1, run drop1(M1) without the test = "F" argument to obtain AIC values for each possible term removal. The <none> row shows the AIC of the current model; all other rows show the AIC after removing the named term. Which term, if removed, gives the lowest AIC? Remove it and call the new model M2.AIC.


14. Continue the AIC-based selection. Run drop1() on M2.AIC, remove the term that yields the lowest AIC, and repeat until no further removal reduces the AIC. Show each step clearly.


15. Both approaches have produced the same final model: lm(ABUND ~ LOGAREA + FGRAZE + LOGAREA:FGRAZE). This will not always be the case. What are the key differences between F-test backward elimination and AIC-based selection, and when might you prefer one over the other?


Validation and reporting


16. Now that you have your final model, carry out model validation using plot() and extract the coefficient estimates and 95% confidence intervals using summary() and confint(). You have already done this for this model in the previous exercise, so this should be quick. What is the biological interpretation of the interaction?


17. Construct a summary table of the models compared during the AIC-based selection, showing AIC and ΔAIC (difference from the best model) for each. Fit each model explicitly using lm() and compute AIC values using the AIC() function.

Note: AIC values from AIC() differ slightly from those returned by drop1() due to a small difference in how the constant term is calculated. This is not a problem, but do not mix values from the two functions in the same table.


18. Write a short paragraph (four to six sentences) suitable for the Methods section of a paper, describing how you arrived at the final model. Your paragraph should include: the starting model and the justification for its structure; which selection approach was used; the criterion for removing terms; the final model; and at least one honest caveat about what the results do and do not show.
